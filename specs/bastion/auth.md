# Bastion: Authentication and Sessions

**Status**: Draft | **Version**: 0.1 | **Part of**: [Bastion Design Spec](README.md)

---

## Session System

Bastion ships with a signed-cookie session system. Sessions are encrypted, tamper-proof, and configurable:

```march
mod MyApp.Config do
  fn session_config() do
    %{
      store: :cookie,                    # or :depot (server-side sessions in Postgres)
      key: "_myapp_session",
      signing_salt: "myapp_signing",
      encryption_salt: "myapp_encrypt",
      max_age: 86400 * 30,              # 30 days
      secure: true,
      http_only: true,
      same_site: :lax
    }
  end
end
```

## Auth Generators

Bastion provides auth scaffolding via Forge generators. Developers choose the auth pattern they want:

```bash
# Cookie-based session auth (traditional web app)
forge gen.auth --strategy session

# Token-based auth (API / SPA)
forge gen.auth --strategy token

# OAuth provider integration
forge gen.auth --strategy oauth --provider github

# Magic link / passwordless auth
forge gen.auth --strategy magic_link
```

Each generator produces:

- Migration files for the users table (via Depot)
- A `MyApp.Auth` module with login/logout/registration functions
- Typed middleware plugs (`require_auth`, `load_current_user`)
- Handler functions for login/registration pages or API endpoints
- Templates for login/registration forms (for session and magic_link strategies)

## Generated Auth Middleware

```march
# Generated by `forge gen.auth --strategy session`
mod MyApp.Auth do
  import Bastion.Middleware
  import Bastion.Conn

  # Typed middleware: loads user from session if present
  fn load_current_user(conn: Conn(WithSession)) -> Conn(WithSession) do
    case conn.assigns.session["user_id"] do
      Some(user_id) ->
        case MyApp.Users.get(conn.assigns.db, user_id) do
          Ok(user) -> conn |> assign(:current_user, Some(user))
          Error(_) -> conn |> assign(:current_user, None)
        end
      None ->
        conn |> assign(:current_user, None)
    end
  end

  # Typed middleware: requires authentication, redirects if not logged in
  fn require_auth(conn: Conn(WithSession)) -> Conn(Authenticated) do
    case conn.assigns.current_user do
      Some(user) -> conn |> assign(:current_user, user)
      None -> conn |> redirect("/login") |> halt()
    end
  end

  fn login(conn, email: String, password: String) -> Result(Conn, AuthError) do
    case MyApp.Users.authenticate(email, password) do
      Ok(user) ->
        conn
        |> put_session("user_id", user.id)
        |> Ok()
      Error(reason) ->
        Error(reason)
    end
  end

  fn logout(conn) -> Conn do
    conn |> clear_session()
  end
end
```

## Secure Cookie Defaults

All cookies set by Bastion use secure defaults:

```march
# Default cookie attributes (applied automatically)
%{
  http_only: true,         # not accessible via JavaScript
  secure: true,            # only sent over HTTPS (auto-disabled in dev)
  same_site: :lax,         # prevents CSRF via cross-site requests
  path: "/",
  max_age: 86400 * 30      # 30 days
}
```

The `secure` flag is automatically set to `false` in development (when running on `http://localhost`) and `true` in production. This is handled by detecting the environment, not by requiring manual config.
