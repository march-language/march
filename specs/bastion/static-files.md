# Bastion: Static File Serving

**Status**: Draft | **Version**: 0.1 | **Part of**: [Bastion Design Spec](README.md)

---

## Built-In Static Server

Bastion serves static files natively, including compiled WASM island bundles. This is important because the framework needs to know about its own island assets for cache-busting, preloading, and development hot-reloading.

```march
# In the router — static files are just a route pattern
fn route(conn, :get, ["static" | path]) do
  Bastion.Static.serve(conn, path,
    from: "priv/static",
    cache_control: "public, max-age=31536000",
    headers: [{"vary", "Accept-Encoding"}]
  )
end
```

## Asset Directory Structure

```
priv/static/
├── css/
│   └── app.css
├── js/
│   └── bastion.js          # Bastion client runtime (island hydration, channels)
├── islands/                 # Compiled WASM bundles (generated by forge build)
│   ├── search_bar.wasm
│   ├── chat_widget.wasm
│   └── manifest.json        # Maps island modules to WASM files + hashes
├── images/
└── favicon.ico
```

## WASM Bundle Management

Forge compiles WASM islands and generates a manifest with content hashes for cache-busting:

```json
{
  "MyApp.Islands.SearchBar": {
    "wasm": "/static/islands/search_bar-a1b2c3d4.wasm",
    "size": 45320,
    "hash": "a1b2c3d4"
  },
  "MyApp.Islands.ChatWidget": {
    "wasm": "/static/islands/chat_widget-e5f6g7h8.wasm",
    "size": 62100,
    "hash": "e5f6g7h8"
  }
}
```

Bastion reads this manifest to generate correct `<script>` and preload tags in rendered HTML.

## Content-Hash URLs and Cache Busting

Static files and WASM island bundles use content-hash URLs (generated by Forge at build time) with long-lived `Cache-Control` headers. This is handled by `Bastion.Static` automatically:

```march
# /static/islands/search_bar-a1b2c3d4.wasm → Cache-Control: public, max-age=31536000, immutable
```

Templates reference hashed asset URLs via a helper:

```march
fn PageLayout(title: String, children: Fragment) do
  ~H"""
  <html>
    <head>
      <link rel="stylesheet" href={Bastion.Static.path("css/app.css")} />
    </head>
    <body>
      {children}
      <script src={Bastion.Static.path("bastion.js")}></script>
    </body>
  </html>
  """
end
# Bastion.Static.path("css/app.css") → "/static/css/app-x9y8z7w6.css"
```

## Development Mode

In development, `forge dev` provides:

- Auto-recompilation of server code on file changes
- Incremental WASM recompilation for changed islands
- Live reload via WebSocket (the page refreshes when server templates change; islands hot-swap when their WASM changes)
- No need for a separate asset pipeline — March is the asset pipeline
